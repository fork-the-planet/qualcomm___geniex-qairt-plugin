# QNN Net Run

[qnn-net-run](https://docs.qualcomm.com/doc/80-63442-10/topic/general_tools.html#qnn-net-run) is a command-line tool for executing QNN models on a specified backend (QnnCpu, QnnGpu, or QnnHtp). During the dev process, `qnn-net-run` can be a good tool to do debug and profiling. 

---

## Command-Line Arguments

### `--backend`

Path to the QNN backend library for model execution.

Backend libraries are located in `<QNN_SDK_ROOT>/lib/<platform>/`. For example:
```
<QNN_SDK_ROOT>/lib/aarch64-android/libQnnHtp.so
```

### `--model`

Path to the model file containing a QNN network.

To compose multiple graphs, provide a comma-separated list of model files:
```
<qnn_model_name_1.so>,<qnn_model_name_2.so>
```

### `--input_list`

Path to a file specifying input files for graph execution.

For multiple graphs, provide a comma-separated list of input list files:
```
<input_list_1.txt>,<input_list_2.txt>
```

Each input list file uses the following format:
```
<input_layer_name>:=<input_layer_path> <input_layer_name>:=<input_layer_path> ...
```

> **Note:** To skip a specific graph, use `__` as a placeholder in the input list:
> ```python
> cmd = [
>     "powershell.exe", QNN_NET_RUN_PATH,
>     "--backend", QNN_HTP_PATH,
>     "--retrieve_context", RETRIEVE_CONTEXT_PATH,
>     "--input_list", f"__,{INPUT_LIST_PATH}",
>     "--config_file", CONFIG_FILE_PATH,
>     "--output_dir", OUTPUT_DIR,
>     "--log_level", "verbose"
> ]
> ```

### `--retrieve_context`

Path to a QNN context binary generated by `qnn-context-binary-generator`.

> **Note:** `--retrieve_context` and `--model` are mutually exclusive. Only one option can be specified at a time.

### `--config_file`

Path to a JSON configuration file for backend extensions, context priority, and graph settings.

For detailed configuration options, refer to the [QNN HTP Backend Extensions](https://docs.qualcomm.com/bundle/publicresource/topics/80-63442-50/htp_backend.html) documentation.

### `--use_native_output_files`

Generates output files in the graph's native data type instead of floating point.

When enabled, two files are generated:
- `output_native.raw` — Raw output in native data type (without dequantization)
- `output_native.raw.json` — Output metadata

Example metadata structure:
```json
{
  "Datatype": "QNN_DATATYPE_UFIXED_POINT_16",
  "Rank": 3,
  "Dimensions": [1, 1, 151936],
  "QuantizationParams": {
    "Type": "SCALE_OFFSET",
    "Scale": 0.000887,
    "Offset": -28530
  }
}
```

### `--use_native_input_files`

Parses input files in the graph's native data type instead of floating point.

### `--profiling_level`

Enables performance profiling. Valid values:

| Level | Description |
|-------|-------------|
| `basic` | Captures execution and initialization time |
| `detailed` | Includes per-op timing (if supported by backend) |
| `client` | Captures only metrics measured by qnn-net-run |

When enabled, a `qnn-profiling-data.log` file is generated in the output directory. Use `qnn-profile-viewer` to analyze the results:

```bash
$QNN_SDK_ROOT/bin/x86_64-linux-clang/qnn-profile-viewer \
    --input_log <path/to/qnn-profiling-data.log> \
    --output <path/to/output.csv>
```

### `--log_level`

Sets the maximum logging verbosity. Valid values: `error`, `warn`, `info`, `debug`, `verbose`.

### `--output_dir`

Specifies the output directory. Defaults to `./output`.
